RegexForAccordance

Search Accordance texts with regular expressions.

System Requirements

  • Mac OSX 10.9 or later
  • Accordance 10.4.5 or later

Getting Started

Choose an Accordance text and enter a verse Range, like "Gen-Mal". Set the search options and the filter options. In the search field, enter a regular expression, and then press return to start searching.

The text is searched line-by-line, one verse per line. Matches do not span verses.

The regular expression matching is provided by Apple's NSRegularExpression class, which implements ICU Regular Expression syntax.

Combining Marks and Composite Characters

A character with combining marks, like Bet with Dagesh (בּ), or Iota with an accent (ί), can be represented two different ways in Unicode: precomposed character or composite character sequence.

As a precomposed character, Bet with Dagesh is a single character: בּ. This character is different from a plain Bet (ב), and a regular expression searching for ב will not match בּ.

As a composite character sequence, the Bet character (ב) is followed by a Dagesh character to form בּ. In this case, ב does match בּ because the Bet and the Dagesh are separate characters.

RegexForAccordance uses composite character sequences by default.

The Search Window

Entering a Range

For the Range, a single book name will do (Gen), or a range of books (Gen-Rev), or specific verses (Gen 1:1-2:7).

Filters

Filters modify the text before the line is searched. Use filters to remove spaces, pointing, diacritics, etc. Click the Filters… button to show the options.

Search Options

If Ignore Case is selected, the search will not be case-sensitive. If Include References is selected, then the verse reference will be part of the search string.

With Include References turned off, the search string for Gen 1:1 is "¶ In the beginning God created the heaven and the earth. ".
With Include References turned on, the search string for Gen 1:1 is "Gen 1:1 ¶ In the beginning God created the heaven and the earth. ".

In either case, the verse reference will be shown in the search results.

Left-to-Right Override

The button to the left of the search field is to toggle Left-to-Right Override.

Mixing right-to-left Hebrew characters with regular expression syntax can get confusing. Use Left-to-Right Override to show the Hebrew letters in reverse order so that you can see their logical relationship with the rest of the regular expression. Although the characters are displayed in reverse order, they will still match the text in their normal right-to-left order.

Left-to-Right Override off: \bאבר\w+ matches אברהם.

Left-to-Right Override on: ‭\bאבר\w+‬ matches אברהם. With the Hebrew characters shown in reverse order, we can see that the word boundary (\b) is followed by Aleph (א), and it will match one or more word-characters (\w+) that follow Resh (ר).

Linking Back to Accordance

Click individual verse references to open them in Accordance. Click the link button above the search results to open all verses in Accordance. Because Accordance has a size limit for verse lists, a long verse list may be divided into sections. These will appear as a popup menu when you click the link button.

The statistics table also has link buttons for each distinct hit.

Statistics Table

Right click the column headers to add or remove columns.

If you select a row in the statistics table, you can use ⌘G and ⇧⌘G to highlight the matching hits in the search results area. You can also copy and paste these rows into another application.
